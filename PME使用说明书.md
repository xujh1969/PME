## 1. 软件简介

### 1.1 产品定位

PME（Portable Markdown Editor）是一款**本地优先**的 Markdown 可视化编辑器，目标体验接近 Typora：你可以像编辑普通文档一样书写 Markdown，同时保留 Markdown 文件的开放性与可迁移性。

PME 的核心理念是：

- **直接编辑 Markdown 文件**，无需先创建项目目录。
- 打开/新建的源文件**尽量保持干净**，不在编辑阶段自动生成 `.pme`、`docs`、`export` 或固定的 `assets` 目录。
- 图片默认保留来源：网络图片保留 URL，本地/粘贴图片使用可显示的临时或数据地址。
- 当你需要把文档和图片一起发给别人时，使用「打包 Markdown」生成副本和 `assets/`，**不修改原始文档**。

### 1.2 主要特点

| 特点 | 说明 |
| --- | --- |
| 所见即所得 | 基于 TipTap / ProseMirror 的可视化编辑，减少 Markdown 源码干扰 |
| 标准兼容 | 编辑阶段保持标准 Markdown，保存后的 `.md` 可被 Typora 等编辑器正常打开 |
| 富文本元素 | 支持标题、列表、任务列表、引用、表格、公式、Mermaid、代码块、图片、视频、链接 |
| 多端运行 | 支持浏览器端运行，以及 Tauri 桌面端（Windows 原生应用） |
| 本地优先 | 文件保存在你自己的磁盘，不依赖云端账号 |

## 2. 安装与运行

PME 有两种使用方式：浏览器端（开发预览）和 Tauri 桌面端（推荐的生产使用方式）。

### 2.1 浏览器端运行（开发预览）

适合开发者或临时预览，需要 Node.js 环境：

```powershell
npm install
npm run dev
```

启动后在浏览器打开：

```
http://127.0.0.1:5173/
```

> 说明：浏览器端受文件系统安全限制，无法像原生应用一样直接读写任意本地路径，完整体验以 Tauri 桌面端为准。

### 2.2 Tauri 桌面端（Windows 应用）

适合普通用户日常使用，需安装 Rust 工具链（含 `cargo`）。

**开发模式（调试）：**

```powershell
$env:PATH="$env:USERPROFILE\.cargo\bin;$env:PATH"
npm run tauri:dev
```

**打包为安装程序：**

```powershell
$env:PATH="$env:USERPROFILE\.cargo\bin;$env:PATH"
npm run tauri:build
```

打包完成后，常见产物位置如下：

```
src-tauri/target/release/pme.exe
src-tauri/target/release/bundle/nsis/PME_0.1.0_x64-setup.exe
src-tauri/target/release/bundle/msi/PME_0.1.0_x64_en-US.msi
```

双击 `PME_0.1.0_x64-setup.exe` 即可安装，之后从开始菜单或桌面快捷方式启动 PME。

### 2.3 启动体验

应用启动时显示深色背景（原生窗口背景为 `#050505`），避免加载时出现明显白屏。

## 3. 界面总览

### 3.1 首页

打开 PME 后首先进入首页，用于进入文件操作：

```
+------------------------------------------------------+
|                                                      |
|              PME                                    |
|     Portable Markdown Editor                         |
|                                                      |
|   [ 新建 Markdown ] [ 打开 Markdown ] [ v 最近文件 ] |
|                                                      |
|                 Hero 背景图                          |
+------------------------------------------------------+
```

- **新建 Markdown**：立即进入编辑器，创建一个尚未保存的空白文档。
- **打开 Markdown**：打开文件选择器，选择 `.md` / `.markdown` 文件。
- **最近文件（下拉箭头）**：显示最近打开过的文档列表，点击即可快速重新打开。

### 3.2 编辑页

进入文档后进入编辑页，布局如下：

```
+------------------------------------------------------+
| 菜单条（文件 / 编辑 / 段落 / 格式 / 视图）             |
+------------------------------------------------------+
| 工具条（图标按钮，分组：文本/格式/段落/插入/视图）      |
+----------------------+-------------------------------+
| 左侧大纲             | 文档标签栏                     |
|                      +-------------------------------+
|                      |                               |
|                      |   TipTap 编辑区               |
|                      |                               |
+----------------------+-------------------------------+
| 状态栏（文件名 / 保存状态 / 字数 / 字符数 / 模式）     |
+------------------------------------------------------+
```

布局约束：

- 菜单条和工具条固定在顶部，状态栏固定在底部，只有编辑区滚动。
- 左侧栏默认显示**大纲**（由标题自动生成）；旧版文件树仅作为兼容能力，不是默认主界面。

### 3.3 状态栏

底部状态栏显示：

- 当前文件名
- 保存状态（已保存 / 未保存）
- 字数
- 字符数
- 当前模式（如编辑、源码等）

## 4. 基础文件操作

### 4.1 新建 Markdown

- 首页点击「新建 Markdown」，或菜单「文件 → 新建 Markdown」，或快捷键 `Ctrl+Alt+N`。
- 立即进入空白文档编辑。
- **第一次保存**时选择保存位置和文件名；保存后，后续 `Ctrl+S` 直接写回该文件。

### 4.2 打开 Markdown

- 首页点击「打开 Markdown」，或菜单「文件 → 打开 Markdown」，或快捷键 `Ctrl+O`。
- 选择本地 `.md` / `.markdown` 文件后直接进入编辑界面。
- 也可以直接通过系统文件管理器双击 `.md` 文件（在已关联 PME 的情况下）由 PME 打开。

### 4.3 最近文件

- 首页点击「最近文件」下拉箭头，或从菜单「文件 → 最近打开」查看。
- 最近打开文件列表显示文件名及完整路径，方便识别。
- 点击列表项在新标签页中打开对应文档，不会替换当前打开的文件。
- 最多显示最近打开的 6 个文件。

### 4.4 保存与另存为

| 操作 | 方式 | 说明 |
| --- | --- | --- |
| 保存 | `Ctrl+S` / 文件 → 保存 | 写回当前文件；首次保存会弹出位置选择 |
| 另存为 Markdown | `Ctrl+Shift+S` / 文件 → 另存为 Markdown | 另选位置和文件名保存副本 |

### 4.5 多文档标签

- 同时打开多个 Markdown 文件时，编辑区上方以**标签页**形式列出。
- 选中标签使用浅色背景 + 清晰边框 + 深色文字，未保存状态用 `*` 或小圆点提示。
- 关闭当前标签：`Ctrl+Alt+W`，或点击标签上的关闭按钮。

### 4.6 关闭未保存提醒

在 Tauri 桌面端，当文档有修改但未保存时关闭窗口，会弹出「保存更改」对话框，提供：

- **保存**：保存修改后关闭。
- **不保存**：放弃修改直接关闭。
- **取消**：返回编辑，不关闭。

## 5. 编辑功能详解

PME 支持丰富的 Markdown 元素，绝大多数操作可通过菜单、工具条或快捷键完成。

### 5.1 文本格式

| 功能 | 工具条/菜单 | 快捷键 |
| --- | --- | --- |
| 加粗 | 格式 → 加粗 | （Markdown `**文本**`） |
| 斜体 | 格式 → 斜体 | （Markdown `*文本*`） |
| 删除线 | 格式 → 删除线 | `Ctrl+Shift+5` |
| 行内代码 | 格式 → 行内代码 | （Markdown `` `代码` ``） |
| 下划线 | 格式 → 下划线 | `Ctrl+U` |
| 文字颜色 | 格式 → 文字颜色 | `Ctrl+Shift+C` |
| 高亮颜色 | 格式 → 高亮颜色 | `Ctrl+Shift+H` |
| 上标 | 格式 | `Ctrl+Shift+Alt+=` |
| 下标 | 格式 | `Ctrl+Shift+Alt+-` |
| 超链接 | 格式 → 超链接 | 弹窗输入链接 |
| 清除样式 | 格式 → 清除样式 | `Ctrl+\` |

> 选择颜色后，文字会自动取消选中状态，方便查看颜色变化效果。

### 5.2 段落与块级元素

通过菜单「段落」或工具条「段落」分组操作：

- **正文 / 标题 1~6 级**：`Ctrl+0` 为正文，`Ctrl+1`~`Ctrl+6` 对应标题级别 1~6。
- **引用**：在段落上应用引用样式（Markdown `> 文本`）。
- **无序列表**：`- 项目`。
- **任务列表**：`- [ ] 待办` / `- [x] 已完成`，适合待办清单。
- **代码块**：`Ctrl+Shift+K`，可指定语言并自动缩进。
- **公式块**：`Ctrl+Shift+M`，用于块级数学公式。
- **分割线**：`---` / `***`。

### 5.3 有序列表编号样式

v0.1.1 起支持自定义有序列表编号样式：

- 默认数字（1, 2, 3）。
- 罗马数字（I, II, III）。
- 字母（a, b, c）。

通过有序列表扩展切换，底层使用 HTML 原生 `<ol type>` 属性。

### 5.4 链接

- 菜单「格式 → 超链接」或在工具条点击链接按钮。
- 在弹窗中输入显示文本与目标 URL，确认后插入为可点击链接（Markdown 格式 `[文本](URL)`）。

### 5.5 标题折叠

标题折叠功能允许你暂时隐藏某个标题下的所有内容，便于在长文档中快速浏览：

- **折叠/展开标题**：将光标置于标题行，点击菜单「段落 → 折叠标题」或工具条对应的按钮。
- **展开所有折叠标题**：点击菜单「段落 → 展开所有折叠标题」。
- **折叠状态保存**：标题的折叠状态会随文档一起保存，下次打开文档时保持之前的折叠状态。
- **大纲交互**：点击大纲中被折叠标题下的子标题时，会自动展开父标题并平滑滚动到对应位置。

### 5.6 目录（TOC）

目录功能自动从文档中的标题生成：

- **插入目录**：点击菜单「插入 → 目录」或工具条对应的按钮。
- 目录会显示当前文档中所有一级到六级标题。
- 导出 HTML 或 PDF 时，目录会自动渲染为完整的列表。

### 5.5 表格

创建表格：

1. 点击工具条「表格」按钮或菜单「插入 → 表格」。
2. 在弹出的中文对话框中输入**行数、列数**，并选择是否包含**表头**。
3. 确认后创建表格。

表格编辑（Typora 风格上下文工具条）：

- 光标进入单元格后，表格上方出现上下文工具条。
- 工具条包含：表格操作入口、左对齐 / 居中 / 右对齐、删除表格、更多操作菜单。
- 「更多操作」支持：
  - 上方插入行 / 下方插入行
  - 左侧插入列 / 右侧插入列
  - 上移该行 / 下移该行
  - 左移该列 / 右移该列
  - 删除行 / 删除列

保存时单元格对齐信息写入 Markdown 表格分隔行，例如：

```markdown
| A | B | C |
| :--- | :---: | ---: |
| 左对齐 | 居中 | 右对齐 |
```

### 5.6 图片

插入图片（菜单「插入 → 图片」或工具条图片按钮）支持四种方式：

- **网络图片**：输入图片 URL，可预览。
- **本地图片**：选择本地图片文件，先预览确认再插入（不会一选就插入）。
- **拖入图片**：直接把图片文件拖入编辑区。
- **粘贴图片**：从网页复制图片后直接粘贴。

图片大小：

- 单击选中图片；**双击**图片可修改大小。
- 大小以「原始宽度的百分比」表示，而非像素宽度；清空大小值表示 Auto（自动）。
- 若设置了比例，保存时输出 HTML `<img>`（带宽度与 PME 私有比例元数据）；否则输出标准 Markdown 图片语法。

### 5.7 视频

插入视频（菜单「插入 → 视频」或工具条视频按钮）：

- **本地视频**：选择本地视频文件插入。
- **拖入视频**：直接把视频文件拖入编辑区。
- 视频支持播放控制（播放、暂停、进度条、音量控制）。
- 视频显示宽度可调整。
- 打包时视频文件会被复制到 `assets/` 目录，确保分享时视频可正常播放。

### 5.8 数学公式（KaTeX 渲染）

- **行内公式**：如 `$E=mc^2$`，写在行内。
- **块级公式**：`$$ ... $$`，独占一段，适合复杂公式。
- 点击公式文字区域触发选择/编辑；点击公式块空白区域不会弹出编辑框。
- 编辑使用多行文本框；清空公式并确认会删除该公式节点。

### 5.9 Mermaid 图表

- 以标准围栏代码块保存：```` ```mermaid ````，内部写 Mermaid 语法（如 `flowchart TD`）。
- 编辑区会将代码渲染为**可缩放的图表**。
- 图表默认按宽度适配；放大超过最佳宽度后可拖拽查看；缩放控制条只在图表被选中时显示。
- 适合画流程图、时序图、状态图等。

### 5.10 代码块

- 插入代码块时可选择编程语言；双击已有代码块或用菜单命令可再次修改语言。
- 代码根据语言**自动缩进**（如识别 `{`、`[`、`(`、Python 冒号、Shell `then/do` 等场景）。
- 代码块使用浅色背景，并提供语法高亮（lowlight）。
- 语言列表支持自定义输入。

## 6. 视图与浏览功能

通过菜单「视图」或工具条「视图」分组操作：

| 功能 | 说明 |
| --- | --- |
| 显示 / 隐藏侧边栏 | 控制左侧栏整体显隐 |
| 显示 / 隐藏大纲 | 控制大纲栏显隐 |
| 搜索 | 调出搜索框，支持中英文精确匹配计数与定位（`Ctrl+F`） |
| 源代码模式 | 切换为 Markdown 源码视图，便于直接查看/修改源文本 |
| 专注模式 | 隐藏干扰元素，专注写作 |
| 显示 / 隐藏状态栏 | 控制底部状态栏显隐 |
| 实际大小 / 放大 / 缩小 | 调整编辑区显示比例，对应快捷键见第 9 节 |

大纲：

- 由文档中的标题自动生成。
- 点击大纲项可定位到对应标题。
- 文档为空或无标题时显示空状态。

搜索：

- 支持中文和英文的精确匹配，显示匹配数量并可逐项定位。
- `Ctrl+F` 打开搜索框，自动聚焦到输入框。
- 查找时自动将光标移动到第一个匹配位置，按「下一个」时移动到下一个匹配位置。
- 选中的匹配文字会自动滚动到显示区域内。

## 7. 对象的选择与编辑

图片、公式、Mermaid 等「对象」的交互方式统一：

- **单击**对象内容：选中该对象。
- 点击页面其他区域：取消选中。
- **Delete / Backspace**：删除选中的对象。
- **双击**对象，或点击对应编辑入口：打开编辑弹窗（如修改图片大小、编辑公式、编辑 Mermaid 代码）。

这样无需先点旁边空白再删除，操作更自然。

## 8. 导出与分享

### 8.1 保存（Markdown）

- 见 4.4 节。保存后的 `.md` 为标准 Markdown，可被 Typora 等编辑器正常打开。

### 8.2 另存为 Markdown

- 「文件 → 另存为 Markdown」（`Ctrl+Shift+S`），另选位置和文件名保存。

### 8.3 导出 PDF

- 菜单「文件 → 导出 PDF」。
- **浏览器端**：保留浏览器打印方式作为降级方案（可能带页眉页脚）。
- **Tauri 桌面端**：通过 Edge headless 直接生成 PDF 文件，去除打印页眉页脚，效果更干净。

### 8.4 打包 Markdown（ZIP 分享）

当你需要把含图片/视频的 Markdown 文档发给别人时使用：

1. 菜单「文件 → 打包 Markdown」。
2. 若文档无图片/视频：直接导出 Markdown。
3. 若含图片/视频：提示你将下载/复制资源（网络图片会被下载，本地图片/视频会被复制）。
4. 生成 ZIP，包含：
   - `文档名.md`（Markdown 副本）
   - `assets/`（图片和视频资源，同一来源只生成一份）
5. 副本中的图片/视频地址改写为包内 `assets/...`。
6. **原始 Markdown 文件不被改写**。

> 这是分享含图片/视频文档的推荐方式：既便携，又不破坏你本地的源文件。

### 8.5 以 HTML 格式打包（ZIP 分享）

当你需要把文档分享给没有安装 Markdown 编辑器的人时使用：

1. 菜单「文件 → 以 HTML 格式打包」。
2. 提示确认打包，显示文档中包含的图片和视频数量。
3. 生成 ZIP，包含：
   - `文档名.html`（转换后的 HTML 文件）
   - `assets/`（图片和视频资源）
4. 解压后可直接用浏览器打开 HTML 文件查看，无需安装 Markdown 编辑器。
5. **原始 Markdown 文件不被改写**。

打包过程中会显示等待动画，提示正在打包资源。

## 10. 已知限制与注意事项

- **Markdown 解析器为轻量实现**，并非完整 CommonMark 解析器。
- **复杂嵌套列表、复杂 HTML、脚注、Front Matter** 等尚未完整支持。
- **Mermaid 缩放比例**目前尚未写入 Markdown 文件。
- **浏览器端**直接读写真实文件路径受安全限制；完整体验以 **Tauri 桌面端**为准。
- **PDF 导出**中的超大 Mermaid 图表暂未做分页/缩放策略，大图可能被裁切，后续版本将优化。
- **拖拽（DragHandle）功能**在部分场景下仍可能受限。
- **打包时**，从不同目录下选择的同名图片/视频会自动生成唯一文件名，不会相互覆盖。

## 11. 常见问题（FAQ）

**Q1：PME 会强制我创建项目目录吗？**  
不会。你可以直接新建或打开单个 `.md` 文件，无需先建项目目录。

**Q2：保存后我的原文件会被加一堆图片文件夹吗？**  
编辑阶段不会。只有当你主动使用「打包 Markdown」时，才会生成包含 `assets/` 的 ZIP 副本，且原始文件保持不变。

**Q3：网络图片会保存到本地吗？**  
编辑阶段网络图片保留原始 URL。只有在「打包 Markdown」时才会把网络图片下载到导出包的 `assets/` 中。

**Q4：保存的 Markdown 能被其他编辑器打开吗？**  
可以。PME 尽量保持标准 Markdown，保存的 `.md` 可被 Typora 等编辑器正常打开。

**Q5：为什么有些复杂 Markdown 显示不正常？**  
当前解析器为轻量实现，对复杂嵌套、复杂 HTML、脚注、Front Matter 支持有限，详见第 10 节。

**Q6：推荐用哪种方式使用 PME？**  
推荐使用 **Tauri 桌面端**（安装后的 Windows 应用），文件读写体验最完整；浏览器端适合临时预览或开发调试。
