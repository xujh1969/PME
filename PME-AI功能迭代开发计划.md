# PME编辑器AI功能迭代开发计划（单任务模式·无右键菜单·仅工具栏/选中文悬浮条入口）

## 核心前置规则（所有AI任务统一遵循，永久生效）

1. **入口规范**：全程不开发任何右键菜单AI入口，仅保留两个固定入口：顶部主工具栏「AI魔法按钮」、选中文本后弹出的悬浮工具条「AI魔法按钮」。

2. **产品底层约束**：完全遵循PME本地优先原则，不自动上传完整文档；AI生成内容仅弹窗预览，用户确认「插入/替换」后才写入文档，支持取消、一键撤销。

3. **语法规范**：所有AI输出内容严格遵循标准Markdown语法，兼容现有解析器、Typora等编辑器，不生成私有语法、不污染原始文件。

4. **交互规范**：所有AI功能统一弹窗交互，包含：输入/选择功能类型、预览生成结果、插入文档、替换选中文本、复制、取消按钮。

5. **配置规范**：统一收纳在软件「文件→配置」中，支持云端API（豆包/通义千问/OpenAI）、本地Ollama离线模型切换、密钥本地加密存储。

6.**AI交互逻辑**：支持流式输出

## 迭代总节奏

按「基础架构→核心文本AI→结构化文档AI→块元素AI（图表/代码/公式）→文档工具AI→离线AI→导出配套AI」顺序，**每次仅执行1个任务**，完成验收后再开启下一个任务。

# 第一阶段：AI基础架构搭建（前置必备，先完成）

### 任务1：开发全局AI入口UI与基础弹窗框架（独立可交付）

**任务目标**：搭建整套AI功能的基础入口和弹窗底座，为后续所有AI功能提供统一载体，无任何业务逻辑，纯架构搭建。

**具体开发需求**：

1. 顶部主工具栏新增**AI魔法棒图标按钮**，固定排版在插入/视图工具组旁，默认常驻显示。

2. 重构选中文本悬浮工具条，新增**AI魔法棒按钮**，选中文本自动显示，取消选中自动隐藏。

3. 点击两个AI按钮，唤起统一风格的AI通用弹窗（空白底座，无功能选项）。

4. AI弹窗标配功能：遮罩层、居右显示、点击空白/ESC关闭、固定窗口大小、适配深浅色主题。

5. 新增全局快捷键 **Ctrl+Shift+A**，一键唤起AI通用弹窗。

6. 不新增任何右键菜单入口，严格遵循产品需求。

**验收标准**：

1. 工具栏、选中文悬浮条AI按钮显示正常、无错位，深浅主题适配完美。

2. 按钮点击、快捷键均可正常唤起/关闭弹窗。

3. 无任何右键AI菜单，界面干净无冗余。

4. 不影响原有所有编辑、视图、文件操作功能。

### 任务2：开发AI配置中心（API/离线模型配置）

**任务目标**：搭建AI参数配置系统，支持云端、离线模型切换，完成本地密钥安全存储。

**具体开发需求**：

1. 在「文件→配置」页面新增独立「AI助手」配置分页。

2. 支持配置：主流大模型API密钥、接口地址、请求超时时间、最大上下文字数。

3. 新增模型切换开关：云端模型 / 本地Ollama离线模型。

4. 配置数据加密存储在本地 `.pme/config.json`，不联网上传、不采集用户配置。

5. 新增总开关：一键开启/关闭所有AI功能，关闭后AI按钮置灰不可点击。

6. 配置实时生效，无需重启软件。

**验收标准**：

1. 配置页面正常展示、参数可正常编辑保存。

2. 模型切换、总开关功能生效正常。

3. 密钥本地加密存储，无明文泄露、无后台上传行为。

# 第二阶段：高频文本AI功能（逐任务开发）

### 任务3：AI选中文本润色/改写功能

**任务目标**：实现最核心的文本AI编辑能力，基于选中内容完成多风格改写。

**具体开发需求**：

1. 仅入口：工具栏AI按钮、选中文本悬浮条AI按钮。

2. 选中文本后点击AI按钮，弹窗展示功能选项：**智能润色、精简压缩、内容扩写、正式化改写、口语化改写**。

3. 调用配置的AI模型，仅上传选中的文本片段，不上传全文。

4. 生成结果弹窗预览，提供三个操作按钮：**替换原文、插入下方、复制到剪贴板、取消**。

5. 生成内容严格保留原文核心语义，输出标准Markdown纯文本，无多余格式。

6. 支持加载动画、请求失败提示（网络错误、密钥错误、超时提示）。

**验收标准**：

1. 所有改写功能正常调用、生成内容合规。

2. 操作后文档格式无错乱，原始文件不自动修改。

3. 错误场景有友好提示，无程序崩溃。

### 任务4：AI文本翻译功能（中英互译\+多语言）

**任务目标**：基于选中文本实现AI智能翻译，适配文档写作场景。

**具体开发需求**：

1. AI弹窗新增「文本翻译」选项，支持中文英文、日文、韩文互译。

2. 自动识别选中文本语种，默认反向翻译。

3. 翻译结果保留原文排版、段落结构，输出标准Markdown文本。

4. 沿用统一预览、替换/插入/复制交互逻辑。

**验收标准**：翻译准确、格式不乱，交互与已有AI功能统一。

### 任务5：AI文档摘要生成功能

**任务目标**：支持对选中文本/整篇文档生成精简摘要。

**具体开发需求**：

1. AI弹窗新增「生成摘要」选项，支持两种模式：选中片段摘要、全文摘要。

2. 支持自定义摘要长度：短（100字）、中（300字）、长（500字）。

3. 全文摘要仅读取本地文档内容，按需分段请求，不一次性上传完整大文件。

4. 摘要内容可一键插入文档顶部/光标位置。

**验收标准**：摘要精炼贴合原文，长文档处理不卡顿，格式标准。

### 任务6：AI智能续写功能

**任务目标**：光标定位处承接前文逻辑，智能续写文档内容。

**具体开发需求**：

1. 无选中文本时，点击AI按钮默认触发「智能续写」功能。

2. 自动读取光标前置段落内容，匹配全文文风、逻辑续写。

3. 支持续写长度选择：短句补充、常规段落、长文拓展。

4. 续写内容预览后插入光标位置，支持撤销。

**验收标准**：续写内容上下文连贯，文风统一，不错乱原有文档结构。

# 第三阶段：结构化文档AI功能（逐任务）

### 任务7：AI新建文档模板生成功能

**任务目标**：空白文档一键生成各类标准Markdown文档结构模板。

**具体开发需求**：

1. 空白编辑页点击AI按钮，弹窗展示模板列表：软件说明书、技术文档、周报、会议纪要、读书笔记、PRD方案。

2. 选择模板后AI自动生成完整层级标题、段落结构、占位内容。

3. 生成内容为标准Markdown，兼容目录、标题、段落原生功能。

**验收标准**：模板结构规范、层级正确，可直接编辑、正常导出。

### 任务8：AI表格智能生成与优化功能

**任务目标**：自然语言生成Markdown表格，优化已有表格数据。

**具体开发需求**：

1. AI弹窗新增「生成表格」选项，支持输入自然语言描述表格行列、内容、用途。

2. 选中已有表格时，AI支持：数据整理、补充行列、内容优化、中英文转换。

3. 生成表格兼容PME原生对齐语法（左/中/右对齐），渲染无异常。

**验收标准**：表格格式标准、渲染正常，可原生编辑修改。

### 任务9：AI全文标题优化与大纲重构

**任务目标**：自动修正全文标题层级混乱，重构标准化文档大纲。

**具体开发需求**：

1. AI弹窗新增「大纲重构/标题优化」功能。

2. 自动扫描全文标题，修正层级错乱、重复标题、不规范命名。

3. 智能优化标题措辞，统一全文风格，生成全新标准化大纲。

4. 预览后一键替换全文标题结构，兼容原生TOC目录功能。

**验收标准**：标题层级规范，大纲预览正常，TOC目录自动同步更新。

# 第四阶段：块元素AI增强（Mermaid/代码/公式，差异化核心功能）

### 任务10：AI Mermaid图表生成与编辑

**任务目标**：自然语言一键生成流程图、时序图等Mermaid图表，优化已有图表。

**具体开发需求**：

1. 光标在空白代码块（选择Mermaid语言）时，点击AI按钮可自然语言生成图表代码。

2. 选中已有Mermaid代码块，AI支持：优化布局、新增逻辑、切换图表类型、补充注释。

3. 生成后实时预览渲染图表，确认后替换代码块内容。

**验收标准**：图表代码标准、渲染正常，支持缩放、拖拽原生功能。

### 任务11：AI代码块辅助功能（注释/重构/转译）

**任务目标**：赋能原生代码块，实现AI代码优化、注释、语言转换。

**具体开发需求**：

1. 选中任意代码块，AI弹窗展示功能：自动添加注释、代码重构优化、语言转换、生成使用示例。

2. 保留原有代码高亮、语言类型、缩进格式。

**验收标准**：代码可正常运行、格式无错乱，注释清晰规范。

### 任务12：AI KaTeX数学公式生成

**任务目标**：自然语言转换为标准行内/块级公式。

**具体开发需求**：

1. AI弹窗新增「生成数学公式」功能，自然语言描述公式需求，AI输出标准KaTeX语法。

2. 选中已有公式，支持简化、变形、补充推导说明。

**验收标准**：公式渲染正常，兼容原生公式编辑功能。
