# PME（Portable Markdown Editor）产品需求文档 PRD

版本：0.1.4  
状态：功能增强版本，已实现标题折叠、视频支持、HTML打包等新功能  
更新时间：2026-07-15

## 1. 产品定位

PME 是一个本地优先的 Markdown 可视化编辑器，目标体验接近 Typora：用户可以像编辑普通文档一样编辑 Markdown，同时保留 Markdown 文件的开放性和可迁移性。

当前产品方向已经从早期的“项目目录管理工具”调整为“通用 Markdown 文件编辑器”：

- 可以直接新建一个 Markdown 文件，不需要先创建项目目录。
- 可以直接打开已有 `.md` / `.markdown` 文件。
- 原始 Markdown 文件尽量保持干净，不在编辑阶段自动创建 `.pme`、`docs`、`export` 或固定 `assets` 目录。
- 图片默认保留来源地址：网络图片保留 URL，本地图片/粘贴图片使用可显示的临时或数据地址。
- 当用户需要把 Markdown 和图片一起发给别人时，使用“打包 Markdown”导出流程生成副本和 `assets/`，不修改原始文档。

## 2. 核心用户场景

### 2.1 编辑单个 Markdown 文件

用户双击或通过软件打开一个外部 Markdown 文件，PME 直接进入编辑界面，左侧显示文档大纲，中心区域显示 Typora 风格的可视化编辑区。

### 2.2 新建 Markdown 文件

用户在首页点击“新建 Markdown”，立即进入空白文档编辑。第一次保存时选择保存位置和文件名。保存后，后续 `Ctrl+S` 直接写回该文件。

### 2.3 插入和粘贴图片

支持：

- 插入网络图片 URL。
- 选择本地图片。
- 拖入图片。
- 从网页复制图片后粘贴。

编辑阶段不强制复制到 `assets/`。需要分享时，通过打包功能扫描文档图片，复制或下载图片到导出包中的 `assets/`。

### 2.4 写作工程文档

PME 已支持标题、列表、任务列表、代码块、表格、公式、Mermaid 图表、图片、链接、搜索和大纲，适合编写技术说明、学习笔记、接口文档和产品方案。

## 3. 当前已实现功能

### 3.1 首页

- 大幅 Hero 背景图。
- “新建 Markdown”主按钮。
- “打开 Markdown”主按钮。
- 最近文件下拉菜单。
- 启动阶段黑色背景，避免明显白屏。

### 3.2 文件能力

- 浏览器端：使用 File System Access API 打开/保存 Markdown 文件。
- Tauri 端：使用原生文件对话框打开/保存 Markdown 文件。
- 支持最近文件记录。
- 支持旧的 workspace 读取逻辑，但新产品入口以 Markdown 文件为主。
- 支持保存、另存/导出 Markdown、打包 Markdown。

### 3.3 编辑能力

- TipTap / ProseMirror 可视化编辑。
- Markdown 读写转换。
- 标题、段落、粗体、斜体、删除线、行内代码、链接。
- 无序列表、任务列表、引用、分割线。
- 代码块，支持语言选择、再次修改语言、低亮高亮、按语言自动缩进。
- 表格，可设置行列数、表头、单元格左/中/右对齐，支持上下左右插入、删除、移动等 Typora 风格上下文操作。
- 数学公式：行内 `$E=mc^2$` 和块级 `$$ ... $$`，使用 KaTeX 渲染。
- Mermaid 图表：使用 ` ```mermaid ` 代码块保存，编辑区渲染为可缩放图表。
- 图片：点击选中，双击修改大小；大小使用原始宽度百分比，Markdown 中以 HTML `img` 保存宽度和 PME 私有比例元数据。
- 视频：支持插入本地视频文件，支持播放控制。
- 搜索：支持中文和英文精确匹配计数与定位，查找时光标自动移动到匹配位置。
- 大纲：从标题自动生成，点击子标题自动展开父标题并平滑滚动。
- 标题折叠：支持折叠/展开标题，折叠状态随文档保存。
- 目录（TOC）：自动从文档标题生成目录，导出时自动渲染。

### 3.4 UI 与交互

- 顶部菜单条和工具条固定。
- 底部状态条固定。
- 文档区域独立滚动。
- 左侧为大纲栏，不再默认显示项目文件树。
- 表格工具条在表格单元被选中时出现。
- 图片、公式、Mermaid 等对象点击后选中，按 Delete / Backspace 删除，双击或命令进入编辑弹窗。
- 所有提示与确认弹窗使用自定义模态框，不使用浏览器 `alert` / `confirm`。
- 视觉风格参考 `notion/DESIGN.md` 的 Notion 风格设计规范。

### 3.5 导出能力

- PDF 导出：
  - 浏览器端保留打印方式作为降级方案。
  - Tauri 端通过 Edge headless 直接生成 PDF 文件，避免打印页眉页脚。
  - 导出时自动渲染目录（TOC）内容。
- 打包 Markdown：
  - 无图片/视频时直接导出 Markdown。
  - 有图片/视频时提示用户需要下载/复制资源。
  - 生成 ZIP，包含 Markdown 副本和 `assets/`。
  - Markdown 副本中的图片/视频地址改写为包内 `assets/...`。
  - 原始 Markdown 文件不被改写。
  - 打包过程显示等待动画。
- 以 HTML 格式打包：
  - 将 Markdown 文档转换为 HTML 格式。
  - 生成 ZIP，包含 HTML 文件和 `assets/`（图片、视频资源）。
  - HTML 文件包含完整样式，解压后可直接用浏览器打开。
  - 打包过程显示等待动画。

## 4. 当前不再采用的早期设计

以下早期设计已废弃或降级为兼容逻辑：

- 不再要求用户先创建项目目录才能编辑文件。
- 不再默认创建 `.pme/config.json`。
- 不再默认创建 `docs/`。
- 不再默认创建 `export/`。
- 不再在编辑时强制把所有图片复制进项目 `assets/`。
- 不再把左侧文件树作为主界面核心区域。

## 5. MVP 成功标准

当前版本的成功标准：

1. 用户可以直接新建 Markdown 并保存到任意位置。
2. 用户可以直接打开已有 Markdown 文件并编辑。
3. 保存后的 Markdown 可以被 Typora 正常打开。
4. 图片、表格、公式、Mermaid、代码块、链接等核心元素可以读写。
5. 用户可以把含图片的 Markdown 打包为可发送 ZIP。
6. Tauri 桌面端可以成功构建 Windows 可执行文件和安装包。

## 6. 后续产品方向

优先级建议：

1. 完善“打开单个 Markdown 文件”在 Tauri 端的真实使用体验，包括最近文件、保存状态和异常提示。
2. 继续增强 Typora 风格表格操作。
3. 增加设置系统：字体、字号、行距、主题、编辑区背景。
4. 增强 PDF 导出模板，特别是 Mermaid 大图分页、缩放和横向页面策略。
5. 增加 Markdown 兼容性测试样本，覆盖更多外部 Markdown 写法。
6. 增加正式的自动更新、版本号和发布流程。
