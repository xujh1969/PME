# PME UI 详细设计

版本：0.1.3  
更新时间：2026-07-15

## 1. 设计目标

PME 的 UI 目标是“首页有视觉冲击力，编辑区接近 Typora，控制区保持 Notion 风格的克制和清晰”。

设计基准：

- 首页：大幅 Hero 背景图，突出 PME 品牌。
- 编辑页：工具条固定、状态栏固定、正文区滚动。
- 文档编辑：所见即所得，减少 Markdown 源码干扰。
- 组件风格：参考 `notion/DESIGN.md`，使用清晰的边框、浅色表面、8px 左右圆角、紫色主按钮、紧凑表单控件。

## 2. 页面结构

### 2.1 首页

首页用于进入文件操作，不展示项目树。

```text
+------------------------------------------------------+
|                                                      |
|              PME                                    |
|     Portable Markdown Editor                         |
|                                                      |
|   [ 新建 Markdown ] [ 打开 Markdown ] [ v 最近文件 ] |
|                                                      |
|                 Hero 背景图                          |
+------------------------------------------------------+
```

交互：

- 点击“新建 Markdown”：立即进入编辑器，创建未保存文档。
- 点击“打开 Markdown”：打开文件选择器。
- 点击箭头：显示最近文件列表，列表不得超出视口。

### 2.2 编辑页

```text
+------------------------------------------------------+
| 菜单条                                               |
+------------------------------------------------------+
| 工具条                                               |
+----------------------+-------------------------------+
| 左侧大纲             | 文档标签栏                    |
|                      +-------------------------------+
|                      |                               |
|                      | TipTap 编辑区                 |
|                      |                               |
+----------------------+-------------------------------+
| 状态栏                                               |
+------------------------------------------------------+
```

约束：

- 菜单条和工具条固定在顶部。
- 状态栏固定在底部。
- 只有编辑区滚动。
- 左侧栏当前只保留大纲；旧文件树只作为兼容能力，不是默认主界面。

## 3. 菜单设计

菜单参考 Typora，但只实现当前可用功能，避免空菜单。

### 3.1 文件

- 新建 Markdown
- 打开 Markdown
- 最近打开（显示最近打开的 6 个文件，带完整路径）

- 另存为 Markdown
- 打包 Markdown
- 以 HTML 格式打包
- 导出 PDF

### 3.2 编辑

- 撤销
- 重做
- 剪切
- 复制
- 粘贴
- 粘贴为纯文本
- 删除
- 查找

### 3.3 段落

- 正文
- 一级到六级标题
- 折叠标题（仅在光标处于标题行时有效）
- 展开所有折叠标题
- 引用
- 无序列表
- 任务列表
- 代码块
- 公式块
- 分割线

### 3.4 格式

- 加粗
- 斜体
- 删除线
- 行内代码
- 超链接
- 清除样式

### 3.5 视图

- 显示/隐藏侧边栏
- 显示/隐藏大纲
- 搜索
- 源代码模式
- 专注模式
- 显示/隐藏状态栏
- 实际大小
- 放大
- 缩小

## 4. 工具条设计

工具条使用图标按钮为主，避免文字按钮堆满界面。

主要分组：

- 文本：标题级别、正文。
- 格式：加粗、斜体、删除线、代码、链接。
- 段落：列表、任务列表、引用、代码块。
- 插入：图片、视频、表格、公式、Mermaid、目录、分割线。
- 视图：源码、搜索、缩放。

所有按钮必须：

- 使用 lucide 图标或清晰的文字缩写。
- 有 tooltip。
- 高度紧凑，文字不得折行。
- 激活状态可见。

## 5. 文档标签页

标签页用于多个 Markdown 文件之间切换。

设计规则：

- 选中标签不使用黑色胶囊。
- 选中态使用浅色背景、清晰边框和深色文字。
- hover 应作用于整个标签区域，而不是只作用于内部文字或关闭按钮。
- 关闭按钮是轻量图标按钮，不抢占标题可读性。
- 未保存状态用 `*` 或小点提示。

## 6. 左侧栏

当前默认显示大纲：

- 从标题节点生成。
- 点击大纲项定位到对应标题。
- 空文档或无标题时显示空状态。

旧文件树约束：

- 只在 workspace 兼容模式显示。
- `assets` 目录默认折叠。
- 不再提供独立“眼睛按钮”打开 assets 浏览区。

## 7. 编辑区

### 7.1 基础排版

- 中文和英文统一使用当前主题字体栈，避免出现中英文混排字体跳变。
- 段落行距与列表行距保持一致。
- 标题输入采用延迟转换：输入 `## 标题` 时，光标仍在该行时不突然吞掉 `##`；离开该行或回车后再显示为真实标题样式。

### 7.2 对象选择

图片、公式、Mermaid 图表等对象：

- 单击对象内容：选中对象。
- 点击页面其他区域：取消选中。
- Delete / Backspace：删除选中对象。
- 双击或点击编辑入口：打开对应编辑弹窗。

这样避免“只能点旁边空白再删除”的不自然操作。

## 8. 表格交互

表格参考 Typora 的上下文工具条：

- 光标进入表格单元格后，在表格上方显示工具条。
- 工具条包含：
  - 表格操作入口。
  - 左对齐、居中、右对齐。
  - 删除表格。
  - 更多操作菜单。
- 更多操作包括：
  - 上方插入行。
  - 下方插入行。
  - 左侧插入列。
  - 右侧插入列。
  - 上移该行。
  - 下移该行。
  - 左移该列。
  - 右移该列。
  - 删除行。
  - 删除列。

创建表格：

- 点击表格按钮后弹出中文对话框。
- 用户输入行数、列数，选择是否包含表头。
- 输入框与窗口边界必须有足够间距。

## 9. 代码块交互

- 代码块使用浅色背景，避免黑色背景与整体 UI 风格冲突。
- 插入代码块时可以选择语言。
- 双击已有代码块或使用菜单命令可以再次修改语言。
- 代码根据语言自动缩进。
- 代码块语言列表必须允许自定义输入。

## 10. 公式交互

- 行内公式：`$E=mc^2$`。
- 块级公式：`$$E=mc^2$$` 或多行 `$$ ... $$`。
- 点击公式文字区域才触发选择/编辑，点击公式块空白区域不弹窗。
- 编辑弹窗使用多行文本框。
- 清空公式并确认时删除该公式节点。

## 11. Mermaid 交互

- Mermaid 以标准 fenced code 保存。
- 编辑区渲染成图表。
- 图表默认按宽度适配，显示区高度根据适配后实际高度计算。
- 放大超过最佳宽度时可以拖拽查看。
- 缩小时显示区高度同步收缩，不应留下大块空白或出现多余垂直滚动条。
- 缩放控制条只在图表被选中时显示。
- 缩放按钮位于图形显示框外的右上方，并固定在该位置。

未来 PDF 导出约束：

- 大图导出 PDF 时不能简单裁切。
- 后续需要在 PDF 导出阶段增加 Mermaid 大图的缩放、横向页面或分页策略。

## 12. 图片交互

插入图片弹窗：

- 本地图片和网络图片是两个紧凑区域。
- 本地图片选择后先预览和确认，不应一选择就插入。
- 网络图片输入 URL 后可以预览。
- 可选设置图片比例，比例是原始宽度百分比，不是像素宽度。
- 不再显示 assets 浏览区。

图片对象：

- 单击选中。
- 双击修改大小。
- 清空大小值表示 Auto。
- 保存时如果设置比例，则输出 HTML `img`，否则输出标准 Markdown 图片语法。

## 13. 弹窗规范

所有弹窗必须：

- 使用自定义模态框。
- 全中文文案。
- 按钮不折行。
- 输入框高度紧凑。
- 输入框和窗口边界有合理 padding。
- 支持 Esc 取消。
- 主按钮和取消按钮视觉层级清楚。

禁止：

- 使用浏览器原生 `alert()`。
- 使用浏览器原生 `confirm()`。
- 弹窗内出现英文按钮如 `OK`、`Cancel`。

## 14. 状态栏

状态栏显示：

- 当前文件名。
- 保存状态。
- 字数。
- 字符数。
- 当前模式。

状态栏固定在底部，不随正文滚动。

## 15. 启动体验

要求：

- `index.html` 和 Tauri 窗口背景均为深色。
- 应用加载前显示深色空背景，减少白屏。
- Tauri 打包版本的原生窗口背景为 `#050505`。

## 16. 标题折叠交互

标题折叠功能允许用户隐藏某个标题下的所有内容：

- **触发条件**：光标位于标题行时，折叠标题按钮才可用。
- **折叠状态标识**：折叠的标题前显示折叠图标（如 `▶`），展开的标题显示展开图标（如 `▼`）。
- **折叠效果**：标题下方的所有内容（直到下一个同级或更高级标题）被隐藏。
- **大纲交互**：点击大纲中被折叠标题下的子标题时，自动展开父标题并平滑滚动到对应位置。

## 17. 视频交互

插入视频弹窗：

- 本地视频选择后先预览和确认，不应一选择就插入。
- 视频显示宽度可调整。

视频对象：

- 单击选中。
- 双击修改大小。
- 播放控制：播放/暂停按钮、进度条、音量控制。
- 保存时视频文件路径保留，打包时复制到 `assets/`。

## 18. 目录（TOC）交互

- 目录自动从文档中的标题生成。
- 显示一级到六级标题，按层级缩进。
- 目录区域有边框和背景，与正文区分。
- 导出 HTML/PDF 时，目录自动渲染为完整列表。

## 19. 等待弹窗

打包等耗时操作时显示等待弹窗：

- 居中显示。
- 包含旋转动画图标。
- 显示提示文字（如"正在打包，请稍候..."）。
- 不可关闭，直到操作完成。

## 20. 设计约束

- 遵循 `notion/DESIGN.md` 中的 token 和组件风格。
- 主要按钮使用紫色主色。
- 普通按钮不使用过度胶囊形状。
- 卡片圆角保持克制。
- 表单控件高度不得过高。
- 所有按钮文字不得折行。
- 不添加与编辑功能无关的大面积装饰。
